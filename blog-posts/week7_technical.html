The DBC Experience
6/29/14

When it comes to databases, most people are probably familiar with SQL databases. However, over the last few years, a new kind of database, called the NoSQL database, has been gaining popularity. What are the differences between the two? What are the benefits of each?

An SQL database is a relational database. It is table-based, meaning data is stored in tables. These tables are made up of columns and rows. A NoSQL database is a non-relational database. Data can be stored as documents, key-value pairs, graphs, and columns. SQL databases use SQL as the standard language for queries. NoSQL databases do not have a standard language (any language can be used including SQL). SQL databases have predefined schemas while NoSQL databases have dynamic schemas which don't need to be predefined. In terms of dealing with increased loads, both SQL and NoSQL databases can be scaled vertically by increasing the power of the hardware (CPU, RAM, etc) on a single server. NoSQL databases can also scale horizontally by increasing the number of servers to deal with larger loads.

Benefits of SQL

  • Standard interface and powerful queries make it the best choice for complex query intensive environment.
  • Reliability and safe guarantee of transactions.
  • Support. SQL databases have been around for a long time and are popular, making it much easier to find free and paid support.

Benefits of NoSQL

  • Scalability. Easy to scale both vertically (increase system resources) and horizontally (adding more servers). Better equipped to deal with increased loads.
  • Best for hierarchical data storage.
  • No predefined schemas make it more flexible. Easier to make changes or add data at any time without disrupting the entire system.
